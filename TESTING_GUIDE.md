# 🧪 Руководство по тестированию системы VERIFIX

## 🎯 Логика смарт-панели "Рекомендуемое действие"

Смарт-панель предлагает действия на основе **статуса сотрудника**:

### 1. Новый сотрудник (найден только в госбазе)
- ✅ Данные подтверждены автоматически
- ❌ Статус: "Не принят"
- **Действия:** "Принять на работу", "Принять на ГПХ", "Сохранить без оформления"

### 2. Дубликат с подтвержденными данными (isVerified: true)
#### 2.1. Если принят на работу (isHired: true):
- **Действие:** только "Сохранить без оформления"

#### 2.2. Если уволен (employmentStatus: terminated):
- **Действия:** "Принять заново", "Принять на ГПХ"

#### 2.3. Если не принят (isHired: false):
- **Действия:** "Принять на работу", "Принять на ГПХ", "Сохранить без оформления"

### 3. Дубликат с неподтвержденными данными (isVerified: false)
- 🔄 **Кнопка под карточкой:** "Обновить данные из госбазы"
- После обновления → проверка на конфликты → переход к сценарию 2

#### 3.1. Если принят на работу (isHired: true):
- **Действие:** только "Сохранить без оформления"

#### 3.2. Если уволен (employmentStatus: terminated):
- **Действия:** "Принять заново", "Принять на ГПХ", "Сохранить без оформления"

#### 3.3. Если не принят (isHired: false):
- **Действия:** "Принять на работу", "Принять на ГПХ", "Сохранить без оформления"

## 📋 Тестовые номера ПИНФЛ для всех сценариев

---

### 1. ✅ **Успешный поиск в госбазе (новый сотрудник)**

**Описание:** Сотрудник найден в государственной базе данных, НЕ найден в локальной системе. Квота тратится. Переход на шаг 2 визарда.

**Тестовые ПИНФЛ:**
- `12345 67890 1234` → **Ахмедов Азиз Рашидович** (Ташкент)
- `98765 43210 9876` → **Каримова Нигора Алишеровна** (Самарканд)

**Результат:**
- ✅ Статус: "Не принят" (зеленый тег)
- ✅ Данные загружены: ФИО, дата рождения, фото, адрес
- ✅ Квота: -1
- ✅ Появляются кнопки действий: "Оформить на работу", "Оформить ГПХ", "Сохранить"

---

### 2. ❌ **Не найден в госбазе**

**Описание:** Данные не найдены по ПИНФЛ. Показывается alert с кнопкой "Добавить вручную". Квота НЕ тратится!

**Тестовые ПИНФЛ:**
- `11111 11111 1111`
- `22222 22222 2222`
- `99999 99999 9999`
- Любой другой ПИНФЛ, которого нет в списке выше

**Результат:**
- ❌ Alert: "Не найдено в госбазе"
- 🔵 Кнопка: "Добавить вручную"
- ⚠️ Квота: без изменений (не тратится!)

---

### 3. 🔴 **Ошибка API/сервера**

**Описание:** Технический сбой при обращении к государственной базе. Квота тратится!

**Тестовый ПИНФЛ:**
- `88888 88888 8888`

**Результат:**
- 🔴 Alert: "Ошибка при обращении к госбазе"
- ⚠️ Квота: -1 (тратится!)
- 🔄 Возможность повторить поиск

---

### 4. 📡 **Нет соединения с сервером**

**Описание:** Отсутствует связь с сервером. Квота НЕ тратится.

**Тестовый ПИНФЛ:**
- `77777 77777 7777`

**Результат:**
- 📡 Alert: "Нет соединения с сервером"
- ⚠️ Квота: без изменений (не тратится!)
- 🔄 Предложение повторить попытку

---

### 5. 🚫 **Квота закончилась**

**Описание:** При quota = 0 блокируется кнопка "Найти". Доступна только ручное создание (если настройка включена).

**Как проверить:**
1. Выполните несколько успешных поисков (используйте ПИНФЛ из пункта 1)
2. Следите за счетчиком квоты (начальное значение: 86/100)
3. Когда квота = 0, кнопка "Найти" станет неактивной

**Результат:**
- 🔒 Кнопка "Найти" заблокирована
- ℹ️ Сообщение: "Квота исчерпана"

## 🎯 Порядок тестирования

### Рекомендуемая последовательность:

1. **Проверка успешного поиска:**
   - Введите `12345 67890 1234`
   - Проверьте отображение данных сотрудника
   - Проверьте статус "Не принят"
   - Проверьте уменьшение квоты

2. **Проверка дубликата (не подтвержден):**
   - Введите `30001 85123 4567`
   - Проверьте предупреждение
   - Проверьте кнопку "Обновить данные"

3. **Проверка дубликата (подтвержден):**
   - Введите `40101 90123 4568`
   - Проверьте информационное сообщение
   - Проверьте дату последнего подтверждения

4. **Проверка "не найден":**
   - Введите `11111 11111 1111`
   - Проверьте alert с кнопкой "Добавить вручную"
   - Убедитесь, что квота НЕ изменилась

5. **Проверка ошибок:**
   - Введите `88888 88888 8888` (ошибка API)
   - Введите `77777 77777 7777` (нет соединения)
