# **PROMPT: Employee Search & Add Wizard — Полное техническое задание**

## **Обзор системы**

Реализовать многошаговый визард для добавления сотрудников в HR систему с интеграцией государственной базы данных. Визард поддерживает поиск по госбазе (с расходованием квоты), автоматическую проверку дубликатов, обновление данных существующих сотрудников, разрешение конфликтов данных и ручное добавление.

---

## **1. Архитектура и Шаги Визарда**

### **1.1 Три основных шага (WizardStep)**
- **`search`** — страница поиска по госбазе (начальный экран)
- **`result`** — страница с результатами поиска и действиями
- **`manual`** — страница ручного ввода данных (без интеграции с госбазой)

### **1.2 Состояния поиска (SearchState)**
```
- idle                        // начальное состояние
- not-found                   // не найдено в госбазе (остается на странице search)
- found-new                   // новый сотрудник найден (переход на result)
- found-duplicate-unverified  // найден дубликат, данные не подтверждены (переход на result)
- found-duplicate-verified    // найден дубликат, данные подтверждены (переход на result)
- found-merge-conflict        // конфликт данных при поиске (переход на result)
- merge                       // конфликт данных после обновления (на странице result)
- updated                     // данные успешно обновлены (на странице result)
- invalid-input               // неверный формат данных (остается на search)
- api-error                   // ошибка API госбазы (остается на search)
- no-connection               // нет соединения с сервером (остается на search)
```

## **2. Система Квот**

### **2.1 Параметры квоты**
- **Месячная квота**: 100 запросов (переменная `maxQuota`)
- **Текущий остаток**: отображается в числовом и процентном виде
- **Визуализация**:
  - Прогресс-бар: зеленый (норма), оранжевый (≤5 запросов), красный (0 запросов)
  - Иконка предупреждения при низком остатке

### **2.2 Операции, расходующие квоту**
1. Поиск в госбазе для **нового сотрудника** (found-new) — **-1 квота**
2. **Обновление данных** существующего сотрудника (update) — **-1 квота**
3. Обнаружение **конфликта данных** (merge) — **-1 квота**
4. **Ошибка API** (api-error) — **-1 квота** (квота списывается даже при ошибке)
5. **Не найдено** (not-found) — **-1 квота**

### **2.3 Операции, НЕ расходующие квоту**
- Поиск сотрудника, который уже есть в локальной базе (verified/unverified)
- Ошибки валидации (invalid-input)
- Проблемы соединения (no-connection)

### **2.4 Блокировки при исчерпании квоты**
- Кнопка "Добавить из госбазы" становится **disabled**
- Кнопка "Обновить данные из госбазы" становится **disabled**
- Доступна только опция "Добавить вручную"

### **2.5 UI Квоты**
```jsx
// Панель квоты (отображается только на странице search)
- Заголовок: "Квота: Март 2026"
- Счетчик: "86 / 100"
- Прогресс-бар с цветовой индикацией
- Подсказка: "Поиск в госбазе и обновление данных расходуют квоту"
- При низкой квоте (≤5): "Осталось 5 запросов"
- При нулевой квоте: "Квота исчерпана" (красным цветом)
```

## **3. Страница Search (Шаг 1)**

### **3.1 Способы поиска (Toggle)**
Два варианта поиска с переключателем:
1. **По ПИНФЛ**:
   - Поле ввода: 14 цифр
   - Плейсхолдер: `12345678901234`
   - Валидация: только цифры, ровно 14 символов

2. **По паспорту + дата рождения**:
   - Поле "Паспорт": серия (2 буквы) + номер (7 цифр), формат `AB 1234567`
   - Поле "Дата рождения": date picker
   - Валидация: обязательны оба поля

### **3.2 Согласие на обработку данных**
- **Checkbox** (обязательный): "Даю согласие на использование персональных данных в соответствии с [политикой конфиденциальности]"
- Кнопка поиска **disabled**, пока checkbox не отмечен

### **3.3 Кнопки действий**
- **"Добавить из госбазы"** (основная, с иконкой Search):
  - Disabled при: `isSearching || isQuotaEmpty || !consentChecked`
  - При нажатии: запуск поиска с индикатором загрузки "Поиск..."

- **"Добавить вручную"** (в правом верхнем углу, ghost):
  - Переход на шаг `manual`
  - Всегда доступна (не требует квоты)

### **3.4 Тестовые ПИНФЛ для отладки**
```javascript
// Новый сотрудник (не в системе):
"11111111111111" → found-new (переход на result)

// Неподтвержденный дубликат:
"31234567890123" → found-duplicate-unverified (переход на result)

// Подтвержденный дубликат:
"41234567890124" → found-duplicate-verified (переход на result)

// Конфликт данных (merge):
"55555555555555" → found-merge-conflict (переход на result)

// Ошибки (остаются на search):
"99999999999999" → invalid-input
"88888888888888" → api-error
"77777777777777" → no-connection

// Любой другой ПИНФЛ:
→ not-found (остается на search)
```

## **4. Логика Поиска (Алгоритм)**

### **Шаг А: Проверка локальной базы**
```
1. Поиск по ПИНФЛ или паспорту в локальной базе
2. Если найдено:
   a. Получить данные из госбазы
   b. Если сотрудник.isVerified === true:
      → Состояние: found-duplicate-verified
      → Toast: "Сотрудник найден в локальной базе (подтвержден)"
      → Переход на страницу result
      → Квота НЕ расходуется
   c. Если сотрудник.isVerified === false:
      → Состояние: found-duplicate-unverified
      → Toast: "Сотрудник найден в локальной базе (не подтвержден)"
      → Переход на страницу result
      → Квота НЕ расходуется
   d. Завершить поиск
```

### **Шаг Б: Поиск в госбазе**
```
3. Если не найдено в локальной базе, запрос к госбазе
4. Если найдено в госбазе:
   a. Проверить на частичное совпадение (конфликт):
      - ПИНФЛ совпадает, паспорт отличается ИЛИ
      - Паспорт совпадает, ПИНФЛ отличается
   b. Если обнаружен конфликт:
      → Состояние: found-merge-conflict
      → Toast: "Обнаружен конфликт данных"
      → Переход на страницу result
      → Квота: -1
   c. Если конфликта нет:
      → Состояние: found-new
      → Toast: "Сотрудник добавлен в систему"
      → Переход на страницу result
      → Квота: -1
```

### **Шаг В: Не найдено**
```
5. Если не найдено ни в локальной базе, ни в госбазе:
   → Состояние: not-found
   → Toast: "Не найдено в госбазе"
   → Остается на странице search
   → Квота: -1
```

Что делать:
• Попробуйте повторить запрос через несколько минут
• Проверьте стабильность интернет-соединения

```

### **5.4 Ошибка: Нет соединения (no-connection)**

Иконка: WifiOff
Заголовок: "Нет соединения с сервером"
Описание: "Не удалось установить соединение с сервером госбазы. Проверьте подключение к интернету и повторите попытку."

Рекомендации:
• Проверьте подключение к интернету
• Убедитесь, что VPN или прокси не блокируют доступ

```

## **11. Критические правила реализации**

1. **Квота расходуется ТОЛЬКО при реальном обращении к госбазе**
2. **Все сценарии (кроме ошибок на search) переходят на result**
3. **Уведомление о подтвержденном дубликате — ЖЕЛТОЕ, не синее**
4. **Конфликт merge может возникнуть на любом этапе**
5. **Ручное добавление доступно всегда (не требует квоты)**
6. **Автоскролл к результатам после завершения операции**
7. **Disabled состояния кнопок при isQuotaEmpty**
